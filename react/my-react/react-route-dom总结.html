Route组件

  • 路由组件props属性，里面有history、location、match三个属性
    • history属性 go(n) go中传入栈索引，跳转对应 goBack() 返回上一级 goForward() 向前一级 push push(path, state) push模式，可以使用3种路由传递值 replace replace(path, state) replace模式，可以使用3种路由传递值
    • location属性 pathname search state
    • match属性 params path url
  • 路由组件是根据导航来渲染
    • 模糊匹配：导航/home/a/b，可以匹配路由/home，但是导航/home，不可以匹配路由/home/a/b
    • 严格匹配：/home/a/b 匹配//home/a/b
    • 多级路由的导航：导航/home/news， 路由通过匹配/home，渲染Home组件，Home组件注册路由，继续匹配Home组件注册的路由 如果严格匹配，匹配/home失败，不会渲染Home，发现无匹配内容，Redirect组件匹配。总结react-route-dom在导航匹配路由， （理解不知道怎么描述）。 在使用Switch组件，Route组件只会匹配一次，应该是在当前Switch组件中匹配一次，如果匹配Route组件中，发现有存在Route组件， 继续匹配二级Route组件路由，如果匹配成功，继续循环操作，如果无匹配内容，寻找Redirect组件，然后结束匹配， 上一级的Switch组件中的Route组件匹配已经结束。(每一次匹配都是按照这个规则 理解)
  • 向路由组件传递参数
    • params参数 导航组件传递1的参数导航 路由组件接受1的参数 参数在props.match.params属性中
    • search参数 导航组件传递1的参数导航 路由组件接受1的参数导航 路由组件接受1的参数
